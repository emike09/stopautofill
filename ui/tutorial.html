Stop Autofill Tutorial

A quick guide to blocking autofill on sites, pages, and specific elements

Quick Start

Stop Autofill works on an opt-in basis. You decide where it applies: this website, this page, or an individual element.

1) Open the extension popup
Click the Stop Autofill icon in your browser toolbar.
2) Choose scope
Toggle This website or This page to enable blocking for that scope.
3) Block a specific element (optional)
Use Block Element to pick one form field/container on the page.

How Scopes Work

Scope determines where your rule applies. Start broad (site/page) or go surgical (element).

This website
Applies to all pages on the current site (recommended if a site is consistently annoying).
This page
Applies only to the current page URL (useful for one-off forms).
Element rule
Applies only to a specific selected element (best when only one part of a page is problematic).

Block Element Picker

The element picker lets you click something on a page (a form, container, or input area) and create a rule for it.

Method A: Popup button
Open the extension popup and click Block Element, then click the element on the page.
Method B: Right-click menu
Right-click the page and choose Stop Autofill: Block element… (you can disable this in Settings).
Tip
If a picker selection is too broad or too narrow, try clicking a parent container or a more specific field.

Default Behavior Settings

These defaults are used when you create new rules.

Disable AutoFill
Prevents address/name/email autofill suggestions.
Disable Password Manager
Discourages browser password suggestions & prompts.
Disable Right-Click Context
Removes the right-click “Block element…” menu item (popup Block Element still works).

Manage Rules (Export / Import)

Use Settings to review totals, export a backup, import rules, or wipe everything.

Export Rules
Downloads a JSON file of your current settings/rules.
Import Rules
Loads a JSON export back into Stop Autofill (great for migrating browsers).
Clear All Rules
Deletes every rule and resets blocking coverage (use with care).

Troubleshooting

A few quick fixes if something doesn’t behave the way you expect.

Rule changes not applying?
Reload the tab after changing rules. Some sites re-render forms dynamically and may need a refresh.
Right-click menu missing?
Check Settings → Disable Right-Click Context. If enabled, the context menu is intentionally hidden.
Element picker feels off?
Try selecting a different container element, or choose the input’s parent wrapper instead of the input itself.
